# Work Skills

Org-specific skills for internal use. These skills may contain proprietary context, internal tooling references, or company-specific workflows.

**Visibility**: `work` — not intended for public sharing.

If this repository is public, consider keeping work skills in a separate private repository and symlinking or submoduling them here.

## Adding a work skill

```
skills/work/
└── my-work-skill/
    ├── SKILL.md
    └── references/
        └── context.md
```

Register the skill in `catalog.json` with `"visibility": "work"`.
